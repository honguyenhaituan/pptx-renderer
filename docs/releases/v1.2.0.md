# v1.2.0 Release Notes

Release date: 2026-06-16

## Summary

v1.2.0 is a backward-compatible minor release focused on large-deck performance,
memory pressure, and rendering fidelity.

The key new capability is opt-in lazy rendering infrastructure. Apps can now combine
windowed list mounting with lazy slide parsing and lazy media decoding so interactive
viewers only parse and decode the slides that are actually needed. Existing integrations
continue to work with the eager defaults.

## Added

- `parseZipLazyMedia()` for indexing media entries during ZIP parsing and decoding media
  bytes on demand.
- `lazyMedia` support on `PptxViewer.open()`, instance `viewer.open()`, and deprecated
  `PptxRenderer`.
- `lazySlides` support for `PptxViewer`, deprecated `PptxRenderer`, and
  `buildPresentation(files, { lazySlides: true })`.
- `materializeSlideNodes()` and `materializeAllSlideNodes()` for consumers that need to
  force deferred slide nodes into the model.
- Performance benchmarking script: `test/perf/render_benchmark.py`.

## Changed

- The public demo now opens decks with recommended ZIP limits, lazy slide parsing, lazy
  media decoding, and windowed list rendering.
- The public demo empty state now has direct upload and sample buttons.
- The public demo now uses the stable list workflow instead of exposing the broken
  list/slide toggle.
- README performance guidance now explains which options to use for uploaded decks,
  long scrollable viewers, media-heavy decks, and export-style workflows.
- Chart renderer internals are split into focused modules for axes, labels, legend,
  palette, series, styles, and post-processing.

## Performance Notes

For interactive large-deck viewers, start with:

```ts
const viewer = await PptxViewer.open(buffer, container, {
  zipLimits: RECOMMENDED_ZIP_LIMITS,
  lazySlides: true,
  lazyMedia: true,
  listOptions: {
    windowed: true,
    batchSize: 4,
    initialSlides: 4,
    overscanViewport: 1.5,
  },
});
```

Local benchmark results showed:

- `lazySlides` reduced model build time by roughly 52-66% on medium and large decks.
- First-window parse + build + render time improved by roughly 16-22% when lazy slide
  parsing was paired with windowed rendering.
- `lazyMedia` reduced initially decompressed media bytes by roughly 72-97% for
  media-heavy windowed viewers.

These are opt-in behavior changes. Eager parsing and full-DOM rendering remain available
for export, print, visual comparison, and workflows that intentionally need every slide
ready in one pass.

## Fixed

- Improved Office-like chart rendering defaults for axes, legends, labels, series
  styling, and mixed chart content.
- Improved text autofit positioning and inherited font/script handling.
- Improved shape rendering for pattern fills, unscaled shadows, arrows, and related
  OOXML style semantics.
- Improved table styling and merged-cell behavior across composite slides.
- Hardened relationship target normalization, namespaced attribute lookup, package media
  alias handling, and OOXML boolean default handling.
- Preserved search and serialization behavior when presentations use deferred slide
  nodes.

## Migration Notes

No migration is required.

To use the new large-deck path, enable `lazySlides`, `lazyMedia`, and windowed list
rendering together. If your integration exports or compares all slides at once, keep the
default eager behavior or explicitly call `materializeAllSlideNodes(presentation)` before
the export step.
